# jinguiSSL

`jinguiSSL` - это библиотека SSL/TLS и криптографии, полностью реализованная на Cangjie. Сейчас проект в первую очередь развивает библиотечные возможности, необходимые для HTTP / SSH, по модели, близкой к BoringSSL, но с приоритетом на нативные API Cangjie.

Рекомендуемая публичная точка интеграции:

```cangjie
import jinguissl.contract.*
```

## Текущее состояние

- Версия: `0.6.0`
- Стадия зрелости: `pre-1.0`
- Тип пакета: `static` библиотека Cangjie
- Публичная интеграционная поверхность: `jinguissl.contract.*`
- Текущий фокус: библиотечные возможности для HTTPS / SSH, а не прикладные бинарные программы

Репозиторий предоставляет криптографические примитивы, работу с сертификатами, TLS/SSH facade и вспомогательные проверки. Он не включает HTTP-фреймворк, SSH-набор приложений или туннельные утилиты.

## Краткая матрица возможностей

- Симметричная криптография: AES-GCM, ChaCha20-Poly1305
- Хеширование и derivation: SHA-256, SHA-384, HKDF
- Случайность: проверка доступности CSPRNG и генерация случайных байтов
- Асимметричная криптография: RSA, ECDHE (P-256 / P-384), X25519, ECDSA, Ed25519
- X.509: разбор PEM / DER, проверка цепочки, проверка hostname, упрощенные API для pinning
- TLS: основы TLS 1.2 / TLS 1.3, record layer, session ticket / cache, key update, exporter
- SSH: X25519 KEX, проверка узла, fingerprint host key и contract helper для transport protection

## Именование

- Человеко-ориентированное имя проекта: `jinguiSSL`
- Имя пакета `cjpm`: `jinguissl`
- Корень import: `jinguissl.*`
- Рекомендуемый стабильный import:

```cangjie
import jinguissl.contract.*
```

Внешним проектам не рекомендуется глубоко зависеть от `jinguissl.crypto.*`, если только они не готовы к дальнейшим изменениям внутренних API в ветке `0.x`.

## Публичная поверхность репозитория

- Исходники: `src/`
- Тесты: `src/jinguissl/tests/`
- Векторы и фикстуры: `testdata/`
- Публичный пример: `examples/phase1-demo/`
- Публичная документация: `docs/`
- Публичный вспомогательный скрипт для упаковки: `tools/cjpm_bundle_finish.sh`

Следующие каталоги в основном используются для локальной совместной работы, справочных материалов или экспериментов и не входят в публичный контракт релиза:

- `_helper/`
- `bridges/`
- `benchmarks/`

## Сборка и проверка

```bash
cjpm build
cjpm test
```

Запускайте их последовательно. Параллельный запуск `cjpm build` и `cjpm test` может конфликтовать из-за общей директории `target/` и давать ложные ошибки.

## Замечание по упаковке

В текущем toolchain Cangjie команда `cjpm bundle` все еще может аварийно завершаться на известном шаге SHA256 indexing уже после создания корректного артефакта. Для этого в репозитории есть вспомогательный скрипт:

```bash
./tools/cjpm_bundle_finish.sh
```

Скрипт запускает `cjpm bundle`, распознает известный сбой после генерации артефакта и дописывает итоговые `sha256` и manifest для валидного `.cjp`.

## Версионирование

`0.6.0` означает, что библиотека уже значительно вышла за пределы phase-1 skeleton, но еще не достигла обещания стабильности 1.0. В линии `0.x` facade `contract` должен оставаться относительно стабильным, а внутренние детали `crypto.*` могут еще меняться.
